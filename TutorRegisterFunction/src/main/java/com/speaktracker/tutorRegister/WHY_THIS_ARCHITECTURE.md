# 아키텍처 디렉터리 구조

## 계층 분리 이유

**models/** - DynamoDB 테이블과 1:1 매핑되는 엔티티. 데이터베이스 스키마를 Java 객체로 표현. 내부 데이터 저장/조회에만 사용.

**dto/** - API 응답 전용 객체. 클라이언트에 노출할 필드만 선택적으로 포함. models를 그대로 반환하면 민감 정보 노출 위험. API 스펙과 DB 스키마를 독립적으로 관리 가능.

**exceptions/** - 타입 안전한 예외 처리. RuntimeException 문자열 대신 명시적 클래스 사용으로 컴파일 타임 검증. HTTP 상태 코드와 에러 코드를 예외 타입에 따라 자동 매핑.

**helpers/** - 멀티 테이블 접근 및 외부 서비스 통합. DynamoDB 4개 테이블, WebSocket, SQS, SES를 추상화. Repository 패턴보다 복잡한 쿼리와 인프라 처리에 적합.
